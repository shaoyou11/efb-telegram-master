# Filter 主界面精简设计

## 目标

避免在主聊天发送 `/filter` 时立即加载并展示全部微信会话，减少等待时间和界面长度。

## 交互

- 主聊天发送 `/filter`：只显示当前静默、完全过滤策略数量，以及“静默”“已过滤”“公众号”“群聊”“联系人”分类入口。
- 点击分类入口：显示该分类下的会话列表，并保留分页、返回和关闭按钮。
- 发送 `/filter 关键词`：直接显示名称中包含该关键词的会话列表。
- 在已绑定的独立话题发送 `/filter`：直接打开当前微信会话的策略设置。
- 主界面不显示全部会话列表，也不提供“全部”入口。

## 分类规则

- 公众号：`vendor_specific.is_mp` 为真的会话。
- 群聊：EFB 群聊类型会话。
- 联系人：排除公众号和群聊后的私聊会话。
- 静默、已过滤：按持久化的会话接收策略筛选。

## 性能与异常

- 优先使用 EFB 已有会话缓存。
- 仅在缓存为空时向从端刷新会话。
- 命令先回复载入状态，避免刷新较慢时完全没有反馈。
- 分类结果为空时显示“没有找到匹配的会话”，不修改现有策略。

## 验证

- `/filter` 主界面不出现具体会话。
- 五个分类入口均能打开正确列表。
- `/filter 关键词` 能显示匹配结果。
- 独立话题内 `/filter` 仍直接设置当前会话。
- 关闭按钮、分页和策略持久化行为保持不变。
